# BizUnit前端业务模块生成提示词

## 模块概述

请根据以下规范生成一个完整的前端业务模块，该模块基于BizUnit架构模式。BizUnit是一个强大的业务单元组件，提供了完整的列表展示、筛选、CRUD操作等功能。

**重要提示**: 本规范基于实际项目中的 Project 组件实现，BizUnit 已经封装了列表、表单、操作等核心功能，开发者主要需要：
1. 定义主组件，使用 BizUnit 作为核心
2. 提供列配置（getColumns）
3. 提供表单内容（getFormInner）
4. 定义操作列表（getActionList）
5. 配置筛选条件（filterList）

## 目录结构

```
src/components/[ModuleName]/
├── index.js                          # 主入口文件，导出所有组件
├── [ModuleName].js                   # 根组件，使用 BizUnit 封装
├── getColumns.js                     # 列配置文件
├── [ModuleName]Form/                 # 表单组件目录（如需要复杂表单）
│   ├── index.js                      # 表单主组件
│   ├── BaseFormInner.js              # 基础表单字段
│   └── OtherFormInner.js             # 其他表单字段
├── Detail/                           # 详情页目录（可选）
│   └── index.js                      # 详情页组件
├── [OtherSubComponents]/             # 其他子组件目录（根据业务需要）
├── style.module.scss                 # 样式文件（按需）
├── doc/                              # 文档和示例目录
│   ├── api.md                        # API文档
│   ├── base.js                       # 主组件示例
│   ├── detail.js                     # 详情示例
│   ├── example.json                  # 示例配置
│   └── summary.md                    # 摘要说明

```

## 核心组件实现规范

### 1. 主入口文件 (index.js)

导出所有子组件：
```javascript

// 根据需要导出其他子组件
```

### 2. 根组件 ([ModuleName].js) - 核心

使用 `BizUnit` 组件作为核心容器：

```javascript
import { createWithRemoteLoader } from '@kne/remote-loader';
import getColumns from './getColumns';
import { Flex } from 'antd';
import { useNavigate } from 'react-router-dom';
import BaseFormInner from './[ModuleName]Form/BaseFormInner';

const [ModuleName] = createWithRemoteLoader({
  modules: ['components-admin:BizUnit', 'components-core:Filter', 'components-core:Global@usePreset']
})(({ remoteModules, baseUrl, apis, onDetail, ...props }) => {
  const [BizUnit, Filter, usePreset] = remoteModules;
  const { SuperSelectFilterItem } = Filter.fields;
  const { ajax } = usePreset();

  return (
    <BizUnit
      {...props}
      apis={{
        create: apis.create[ModuleName],
        list: apis.[moduleName]List,
        save: apis.save[ModuleName]  // 可选，如果 BizUnit 需要编辑功能
      }}
      getColumns={() => getColumns({ onDetail: ({ colItem }) => onDetail(colItem) })}
      getFormInner={() => (
        <Flex vertical gap={10}>
          <BaseFormInner />
          {/* 其他表单部分 */}
        </Flex>
      )}
      name="[moduleName]"  // 业务名称，用于生成按钮文案等
      getActionList={({ data, ...props }) => [
        // 定义操作按钮
        {
          ...props,
          name: 'detail',
          children: 'Detail',
          onClick: () => onDetail(data)
        },
        {
          ...props,
          name: 'edit',
          children: 'Edit',
          onClick: () => { /* 编辑逻辑 */ }
        },
        {
          ...props,
          name: 'delete',
          children: 'Delete',
          confirm: true,
          message: 'Are you sure to delete?',
          onClick: async () => { /* 删除逻辑 */ }
        }
      ]}
      filterList={[
        [
          <SuperSelectFilterItem
            name="status"
            label="Status"
            single
            options={[
              { label: 'Online', value: 'online' },
              { label: 'Offline', value: 'offline' }
            ]}
          />
        ]
      ]}
      options={{
        bizName: '[ModuleName]',
        tableProps: {
          pagination: {
            currentName: 'page',
            pageSizeName: 'size'
          }
        },
        keywordFilterName: 'name',
        keywordFilterLabel: '[ModuleName] Name',
        getFilterValue: filterValue => ({ params: filterValue })
      }}
    />
  );
});

export default [ModuleName];
```

**BizUnit 核心属性说明**：

| 属性 | 说明 | 类型 |
|------|------|------|
| apis | API配置对象 | `{ create, list, save, detail, remove }` |
| getColumns | 返回表格列配置的函数 | `() => ColumnType[]` |
| getFormInner | 返回表单内容的函数 | `() => ReactNode` |
| getActionList | 返回操作按钮列表的函数 | `({ data, ...props }) => ActionType[]` |
| filterList | 筛选条件配置 | `JSX.Element[][]` |
| name | 业务名称，用于生成默认文案 | `string` |
| options | 配置选项 | `Object` |

### 3. 列配置 (getColumns.js)

导出 `getColumns` 函数，定义表格列：

```javascript
const getColumns = ({ onDetail }) => {
  return [
    {
      name: 'name',           // 数据字段名
      title: '[ModuleName] Name',  // 列标题
      onClick: onDetail,      // 点击事件
      type: 'mainInfo'        // 列类型
    },
    {
      name: 'status',
      title: 'Status',
      type: 'tag',
      valueOf: item => ({      // 用于枚举值的映射
        isEnum: true,
        moduleName: 'onlineStatus',
        name: item.status
      })
    },
    {
      name: 'created_at',
      title: 'Created At',
      type: 'datetime'
    },
    {
      name: 'description',
      title: 'Description',
      type: 'description'
    }
  ];
};

module.exports = getColumns;
```

**列类型支持**：
- `mainInfo`: 主信息列，可点击
- `tag`: 标签列
- `date`: 日期列
- `datetime`: 日期时间列
- `description`: 描述列

### 4. 表单组件 (FormInner/index.js 或 BaseFormInner.js)

使用 `FormInfo` 组件定义表单字段：

```javascript
import { createWithRemoteLoader } from '@kne/remote-loader';

const BaseFormInner = createWithRemoteLoader({
  modules: ['components-core:FormInfo']
})(({ remoteModules, ...props }) => {
  const [FormInfo] = remoteModules;
  const { Input, DatePicker, TextArea, Select } = FormInfo.fields;

  return (
    <FormInfo
      {...props}
      title="Basic Information"
      column={1}
      list={[
        <Input name="name" label="[ModuleName] Name" rule="REQ LEN-0-100" />,
        <TextArea name="description" label="Description" rule="LEN-0-500" />,
        <DatePicker name="date" label="Date" rule="REQ" />,
        <Select
          name="status"
          label="Status"
          rule="REQ"
          options={[
            { label: 'Online', value: 'online' },
            { label: 'Offline', value: 'offline' }
          ]}
        />
      ]}
    />
  );
});

export default BaseFormInner;
```

**FormInfo 字段类型**：
- `Input`: 文本输入
- `InputNumber`: 数字输入
- `TextArea`: 文本域
- `Select`: 下拉选择
- `DatePicker`: 日期选择
- `DatePicker.RangePicker`: 日期范围选择
- `Switch`: 开关

**验证规则**：
- `REQ`: 必填
- `LEN-0-100`: 长度限制（0-100字符）

## 操作按钮配置

在 `getActionList` 中定义操作按钮：

```javascript
getActionList={({ data, ...props }) => [
  {
    ...props,
    name: 'detail',
    children: 'Detail',
    onClick: () => onDetail(data)
  },
  {
    ...props,
    name: 'edit',
    children: 'Edit',
    hidden: data.status === 'locked',  // 条件显示
    onClick: () => { /* 编辑逻辑 */ }
  },
  {
    ...props,
    name: 'online',
    children: 'Online',
    confirm: true,  // 显示确认弹窗
    isDelete: false,
    message: 'Are you sure to online this [moduleName]?',
    onClick: async () => {
      const { data: resData } = await ajax(
        Object.assign({}, apis.save[ModuleName], {
          urlParams: { id: data.id },
          data: { status: 'online' }
        })
      );
      if (resData.code !== 0) return;
      props.onSuccess();  // 刷新列表
    },
    hidden: data.status === 'online'
  },
  {
    ...props,
    name: 'offline',
    children: 'Offline',
    confirm: true,
    message: 'Are you sure to offline this [moduleName]?',
    onClick: async () => {
      const { data: resData } = await ajax(
        Object.assign({}, apis.save[ModuleName], {
          urlParams: { id: data.id },
          data: { status: 'offline' }
        })
      );
      if (resData.code !== 0) return;
      props.onSuccess();
    },
    hidden: data.status === 'offline'
  }
]}
```

**操作按钮属性**：
- `name`: 操作名称
- `children`: 按钮文案
- `onClick`: 点击处理函数
- `confirm`: 是否显示确认弹窗
- `message`: 确认弹窗提示信息
- `hidden`: 是否隐藏（布尔值或函数）
- `isDelete`: 是否为删除操作

## 生成要求

1. 使用 BizUnit 作为核心组件
2. 所有组件使用 `createWithRemoteLoader` 加载依赖
3. 实现完整的CRUD功能（通过 BizUnit 配置）
4. 提供完整的文档示例
5. 代码风格统一，使用ESLint
6. 注释清晰，易于维护
7. 支持响应式设计

## 与原规范的差异

| 原规范 | 当前规范（基于实际项目） |
|--------|----------------------|
| 需要实现 List、Actions、FormInner 等多个独立组件 | BizUnit 已封装这些功能，只需配置 |
| 需要手写路由配置 | BizUnit 内部处理，无需路由 |
| 需要 withLocale 国际化封装 | BizUnit 内部处理国际化 |
| 需要 ChildrenRouter | 无需路由组件 |
| 复杂的目录结构 | 简化的目录结构，聚焦业务 |
